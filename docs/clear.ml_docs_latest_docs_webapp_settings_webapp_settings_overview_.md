Use the **Settings** page to manage your ClearML account and configure your workspace settings.

To navigate to the Settings page, click the ![Profile button](https://clear.ml/docs/latest/icons/ico-me.svg)
button in the top right corner of the web UI screen, then click **Settings**.

The Settings page consists of the following sections:

- [User Settings](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_profile):

  - [Profile](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_profile#profile) \- You basic user information
  - [Configuration](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_profile#configuration) \- Control general system behavior settings and input storage access credentials
  - [Workspace](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_profile#workspace)
    - [ClearML credentials](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_profile#clearml-api-credentials) \- Create client credentials for ClearML and ClearML Agent to use
    - [Configuration vault](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_profile#configuration-vault) (ClearML Enterprise Server) - Define global ClearML client settings
      that are applied to all ClearML and ClearML Agent instances (which use the workspace's access
      credentials)
- Administrator Settings:
  - [Administrator Vaults](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_admin_vaults) (ClearML Enterprise Server) - Manage user-group level configuration
    vaults to apply ClearML client settings to all members of the user groups
  - [Users & Groups](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_users) \- Manage the users that have access to a workspace
  - [Access Rules](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_access_rules) (ClearML Enterprise Server) - Manage per-resource access privileges
  - [Identity Providers](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_id_providers) (ClearML Enterprise Server) - Manage server identity providers
  - [Resource Configuration](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_resource_configs) (ClearML Enterprise Server) - Define the available resources and the way in which they
    will be allocated to different workloads
  - [Application Gateway](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_app_gw) (ClearML Enterprise Server) - Monitor all active application
    gateways, and verify gateway functionality
  - [Billing & Usage](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_usage_billing) (ClearML Hosted Service) - View current billing details and usage information
  - Storage
    - [Volumes](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_storage_volumes) (ClearML Enterprise Server) - Define and manage storage configurations that can be assigned to projects
    - [Cleanup](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_storage_credentials) \- Configure storage provider access credentials to
      enable ClearML to delete artifacts stored in cloud storage when tasks and models are deleted
  - [UI Customization](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_ui_customization) (ClearML Enterprise Server) - Configure default naming templates for cloned tasks and new Hyper-Dataset versions
  - [Analytics](https://clear.ml/docs/latest/docs/webapp/settings/webapp_settings_analytics) (ClearML Enterprise Server) - Monitor usage and spend over time
